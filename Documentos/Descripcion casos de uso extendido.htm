RF-6(Modificar pedido)

Versión

1- 21/00/2016

Autores

Vendedor(transferencista)

Fuentes

Documento IEEE830

Objetivos asociados

Generar formatos que agilicen el proceso de cargue de pedidos de manera más eficiente.

Descripción

Este caso de uso le permite al usuario vendedor  modificar un pedido en caso de que este sea cancelado o se hagan cambios dentro del tiempo determinado.

Precondición

El usuario debe tener un rol pre- definido (vendedor-transferencista) y un pedido creado antes de poder modificarlo  en el sistema.

Secuencia

Normal

Paso

Acción

1

El usuario se autentica en el sistema.

2

El sistema verifica la validación del usuario

3

El sistema predecirá el rol al que pertenece cada usuario y mostrará la pantalla inicial correspondiente

 

4

El usuario (Transferencista) agregara el pedido teniendo en cuenta el distribuidor al que corresponde.

5

 El usuario (transferencista) se dirigirá directamente al pedido específico  donde se desplegara un menú mostrando  distintas opciones.

6

El vendedor ingresara en la opción “modificar pedido” donde mostrara una ventana con las opciones para poder realizar la acción.

7

El usuario (Transferencista) modificara el pedido cuando sea necesario, teniendo en cuenta las  fechas límites.

Escenario Alternativo

4

 En caso de no estar vinculado el cliente en el sistema  el usuario (transferencista) creara un nuevo cliente en el sistema para retornar al paso 4.

Pos condición

El usuario (transferencista) ha guardado correctamente los datos del pedido.

Excepciones

Paso

Acción

1

En caso de que el tiempo mínimo de modificación del pedido se hubiese agotado no se podrá modificar ni cancelar el pedido.

Rendimiento

Paso

Objetivos asociados

Generar formatos que agilicen el proceso de cargue de pedidos de manera más eficiente.

Descripción

 

Esta funcionalidad está hecha para crear las órdenes de pedidos que serán procesadas por los Usuarios Distribuidores.

Precondición

El usuario debe contar con los permisos para realizar esta acción, eso significa que su perfil corresponde al de transferencista, el usuario solo podrá realizar esta acción cuando su sesión se encuentre iniciada.

Cada Pedido deberá ser procesado después de su creación por el usuario Distribuidor para ello debe ser asociado al pedido durante su creación y entonces tanto el Distribuidor como el cliente debe existir previamente en el sistema de no ser así el pedido no podrá ser terminado.

Secuencia

Normal

Paso

Acción

1

El usuario transferencista desde su sesión iniciada selecciona la opción Crear Pedido en su interface y se desprenderán varias acciones a manera de menú

2

Las opciones de la funcionalidad Crear pedido son:

Nuevo pedido, Continuar Pedido y Consultar Pedidos Abiertos.

El usuario debe seleccionar la opción Nuevo Pedido o Continuar pedido si desea terminar una pedido que se encuentra incompleto.

3

En el caso de un nuevo pedido se diligencia el formulario del pedido y en el caso de continuar pedido llenaremos los campos que estos incompletos.

4

Antes de finalizar el pedido se debe seleccionar el distribuidor y cliente al que están asociados o de lo contrario el sistema no le permitirá terminar pero le dará la opción de guardar como pedido incompleto.

Escenario Alternativo

3

En el caso de ingresar un pedido sin distribuidor el sistema le dará la opción de guardar el pedido como: “sin terminar” para ser completado más adelante desde la opción continuar pedido y regresar al paso 2.

Postcondición

Después de la creación del pedido esta tomara un estatus de enviado y quedara en espera su confirmación por parte del distribuidor.

Excepciones

Paso

Acción

1

En caso de no tener los productos solicitados en el inventario la creación del pedido se cancelara.
